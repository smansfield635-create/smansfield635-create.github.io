Hybrid Collapse Rule

Doctrine-only classifier. Defines output classes and trigger rules under windowed evaluation.

Output classes

The classifier produces exactly one class for a declared epoch and declared ruleset.

C0 · PASS
C1 · FRACTURE
C2 · PHASE TRANSITION
C3 · IMPOSSIBILITY

Bit order (canonical)

Bits are declared and evaluated in fixed order. No reordering allowed without version bump.

Bits: [I P G S R L Y F] ∈ {0,1}^8

Core triplet

The core stability triplet is the only admissible core sum used by this ruleset.

K(t) = I(t) + S(t) + R(t) ∈ {0,1,2,3}

Window lock

Default sustained window length is W=8. Any change requires a version bump.

W = 8 Counts are evaluated on sliding windows. No weighting. No compensation. No smoothing beyond window counts.

Rule set (Hybrid v1)

Rules are applied in order. First triggering rule determines the output class.

R0 — INSTANT IMPOSSIBILITY (HARD KILL) Trigger if ANY holds at any step t: R0.1 K(t) = 0 OR R0.2 (I(t)=0 AND S(t)=0) OR R0.3 (I(t)=0 AND R(t)=0 AND P(t)=0) => OUTPUT = C3 R1 — PHASE TRANSITION (SUSTAINED CORE FAILURE) CF(t) := [K(t) < 2] If CF occurs in ≥ ceil(0.75*W) steps within any sliding W-window (default ≥6 of 8) => OUTPUT = C2 R2 — FRACTURE (GATE/PARTITION BREACH WITH CORE PRESENT) FF(t) := [(P(t)=0 OR G(t)=0) AND K(t) >= 2] If FF occurs in ≥ ceil(0.50*W) within any sliding W-window (default ≥4 of 8) => OUTPUT = C1 R3 — LENS/PAYLOAD FAIL (SECONDARY, NON-LETHAL) SF(t) := [(L(t)=0 OR Y(t)=0) AND K(t) >= 2] If SF occurs in ≥ ceil(0.75*W) within any sliding W-window (default ≥6 of 8) => OUTPUT = C1 NOTE: R3 alone never yields C2/C3. R4 — PASS If none of R0/R1/R2/R3 triggers => OUTPUT = C0

Scope binding

This rule applies only inside a declared scope: epoch bounds + active lanes + active anchors + receipt-backed evidence stream.

If the scope changes, the run is a new epoch or a segment break. No retroactive edits.
