How to Read Gauges

A gauge is a diagnostic reading. It usually answers four questions: what was checked, whether it was found, whether it was readable, and what action should happen next.

Common readings

  • PASS means the component is present and the checked condition is acceptable.
  • HOLD means the component is not failed, but more evidence is needed.
  • BLOCKING=true means the next action cannot lawfully proceed until the block is resolved.
  • BLOCKING=false means the chamber is not blocked, but there may still be a next check.
  • UNKNOWN means the chamber cannot prove the value from current evidence.

What to look for

Look for FIRST_BLOCKING_COMPONENT, NEXT_FILE, NEXT_ACTION, and whether protected-system flags remain false.

Receipt Levels

The receipt deck separates readings by level so a reader can distinguish load checks, component checks, direct execution, synthesis, and deep archive.

  • Level 1 — Basic load and presence checks.
  • Level 2 — Component receipts and gauge summaries.
  • Level 3 — Direct execution checks such as North, East, South, West, HTML Receiver, and Canvas Boundary.
  • Level 4 — Synthesis and Next Lawful Move.
  • Level 99 — Deep Archive for the full diagnostic state.

Chamber Roles

The diagnostic chamber separates roles so evidence does not collapse into one blended authority.

  • North — final diagnostic arbitration and rail synthesis.
  • East — intake, admission, and route-surface reading.
  • South — proof output, sidecar reading, and receipt organization.
  • West — rendered target, endpoint-family, and boundary reading.
  • LabWest — construct alignment gate; separate from West Diagnostic.
  • Canvas Boundary — protection line around production drawing and release authority.

Direct Checks

A direct check resolves the authority live when the button is clicked. This prevents the chamber from trusting a stale object captured at page load.

Why this matters

If a file was renewed but the receipt still shows an old contract, likely causes include cache, script order, stale reference, or a later script overwriting the public alias.

West example

If HEARTH.diagnosticWest is found but WEST_DIAGNOSTIC_DIRECT_METHOD=NONE, the object exists but the chamber cannot find a callable method. If the live object has runDiagnostic but the receipt still says NONE, the receipt deck is reading stale state and must be renewed.

Blocking vs Non-Blocking

Blocking means a required component is missing, malformed, unsafe, or unreadable in a way that prevents the next lawful action. Non-blocking means the page can continue, even if a direct check is still recommended.

  • FIRST_BLOCKING_COMPONENT=NONE means no hard blocker is currently proven.
  • FIRST_BLOCKING_REASON=NONE means no hard blocking reason is currently proven.
  • NEXT_ACTION can still request another check even when BLOCKING=false.

Trusted Receipts

Trusted receipts are the receipt layers the synthesis is allowed to use for the current Next Move. A receipt can be available without being trusted yet. Direct receipts are often run after synthesis asks for them.

When DIRECT_RECEIPTS_AVAILABLE includes a number but DIRECT_RECEIPTS_RUN=NONE, the chamber is saying direct checks exist but have not been run in the current synthesis path.

Next Lawful Move

The Next Lawful Move is the chamber’s focused recommendation. It should point to one next file and one next action. It is not a production authorization.

How to read it

  • NEXT_FILE is the file or route surface to inspect, run, or renew.
  • NEXT_ACTION is the immediate action.
  • DO_NOT_TOUCH lists protected systems that must not be changed.

Protected Systems

This chamber does not authorize production mutation, Canvas build, Canvas release, controls mutation, or runtime restart. Those systems remain protected unless a separate lawful contract explicitly authorizes them.

  • PRODUCTION_MUTATION_AUTHORIZED=false
  • CANVAS_BUILD_AUTHORIZED=false
  • CANVAS_RELEASE_AUTHORIZED=false
  • CONTROL_MUTATION_AUTHORIZED=false
  • RUNTIME_RESTART_AUTHORIZED=false

Glossary

  • Authority — the public object or route surface the chamber can inspect.
  • Alias — a public path such as HEARTH.diagnosticWest.
  • Receipt — machine-readable proof that a component or check exists.
  • Direct method — a callable function such as runDiagnostic() or getReport().
  • Light scan — a quick overview of major chambers and next action.
  • Deep archive — the full state dump for the receipt deck.
  • Sidecar — a diagnostic adapter that reads evidence without owning production authority.
  • Boundary — a protection line that prevents diagnostic checks from becoming production mutations.
