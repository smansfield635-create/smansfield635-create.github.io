Problem

The problem addressed here is structural, not cosmetic: systems fail when execution authority, observation, and registry are mixed or ambiguous.

Observed Failures

  • Hidden coupling between execution and observation.
  • Implicit state mutation without declaration.
  • Non-deterministic discovery due to inconsistent naming.
  • UI surfaces acting as authorities.

Root Causes

  • Lack of a single execution surface.
  • No enforced registry prefix.
  • Observer logic running in the same context as writers.
  • Navigation not aligned to filesystem truth.

Constraints Required

  • Prefix Law: discovery by ENGINE::.
  • Authority Law: Control Room writes; others read.
  • Separation Law: observer-only dashboards.
  • Filesystem Law: URL == folder == index.html.

Resolution Path

Introduce a single Control Room as the authoritative executor, enforce a global discovery prefix, and constrain all other pages to observer or viewer roles.
