Core Invariants

Non-overridable constraints. Scope precedes authority. No compensation by weighting.

Definition

Invariants are constraints that must remain true under a declared scope and declared measurement rules.

Non-negotiable set

If any invariant fails, the system is inadmissible under that scope—no balancing or compensation is allowed.

  • Non-contradiction under identical scope and evidence
  • Manifest/measurement lock (no semantic swaps)
  • Evidence chain integrity (receipt continuity)
  • Non-retroactivity (new epoch on correction)

Operational implication

Any downstream product or gauge inherits these invariants as hard boundary conditions.

Bindings

These links route to implementation surfaces. They are not part of doctrine itself.
