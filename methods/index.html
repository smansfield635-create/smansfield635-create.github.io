Methods

Constraint-first diagnostic system. Evidence-only. No intent inference. This page defines invariant laws, categorical structure, and audit limits.

Core Loop

CHECK → RECEIPT → LOG → SEARCH → PROOF
Results exist only when traceable.

Scientific Findings

S1 — Provability from structure. Execution admissible only when constraints are satisfied and outcomes trace.
S2 — Receipts block silent change. No receipt, no state transition.
S3 — Drift is observable. Detected via violated constraints or bypassed gates.
S4 — Trajectory is bounded. Tightening constraints changes reachable futures.
S5 — Invariants preserve compression. Loss of invariants produces false positives.

Categorical Findings

C1 — Execution band (0–7). Formation → constraint → boundary → executable minimum → stability → projection → feedback → completion.
C2 — Visibility band (9–13). Visibility → closure → extension → higher-order execution → re-stabilization.
C3 — Closure rule. Additions fail unless explicit exceptions are introduced.

Independent Findings

I1 — Reproducible classification. Independent observers converge on the same state.
I2 — Verifiable outcomes. Verified via receipts and append-only logs.
I3 — Objective drift detection. No narrative inference required.
I4 — Audit without authority. Validity governed by artifacts, not identity.

Audit Limits

No inference of intent, motive, personality, or internal cognition. Mixed-domain claims are rejected. Missing receipts invalidate results.

Cyclical Law

Definition.

The Cyclical Law defines a universal mechanism of operation across domains. A system advances by repeatedly executing a closed loop of evaluation, reduction, contrast, validation, and regeneration until a stable boundary or blocking wall is reached.

The law applies to living systems, engineered systems, research programs, and institutions. It governs function, not belief, intent, or narrative.

Cyclical Law Run.

A Cyclical Law Run is a bounded execution of the cycle below, performed until one of the terminal conditions is met.

  • Reduce — Remove duplication, collapse noise, retain invariants.
  • Contrast — Test distinctions, boundaries, and exclusions.
  • Validate — Check against evidence, receipts, and artifacts.
  • Improve — Increase clarity, stability, or explanatory power.
  • Regenerate — Re-enter the cycle at the next admissible state.

Hurdle vs Wall.

A hurdle is a temporary obstruction that can be cleared through research, restructuring, or external linkage without invalidating the cycle.

A wall is a hard boundary where no admissible advance exists without violating evidence rules, constraints, or scope. Walls terminate the run.

Terminal Conditions.

  • Stable completion is reached (cycle closes cleanly).
  • A wall is detected (no lawful advance possible).
  • The system collapses to baseline (7 → 0 reset).

Structural Alignment.

The cyclical law aligns with the structural cycle: 0–7 as execution formation and 9–13 as visibility, closure, and re-stabilization.

Baseline is treated as 7 ≡ 0 when regeneration is required.
