Shared instrument branch

Assets

The reusable layer under the pages.

The Assets Branch holds the shared interface instruments: diamond controls, globe boxes, route surfaces, and reusable UI standards. Pages should consume these instruments instead of rebuilding them locally.

Branch purpose

Assets prevent local page drift.

Generation 2 should not require every page to reinvent bubbles, diamonds, globes, or motion. The shared assets layer gives each page a stable source of visual and runtime truth.

CSS authority /assets/ui.css

Owns shared visual contracts: panels, diamonds, bubbles, route cards, globe box styling, and responsive density.

CSS_AUTHORITY=/assets/ui.css
Runtime authority /assets/universe_ui.js

Owns reusable globe mounting, shared runtime receipts, and page-level instrument activation.

RUNTIME_AUTHORITY=/assets/universe_ui.js
Page contract Consume, do not rebuild.

Pages should request instruments through data attributes instead of duplicating large local implementations.

RULE=CONSUME_SHARED_ASSETS
Reusable globe instrument

One globe system, many page placements.

The shared globe box lets any page mount a real-Earth instrument with a size, theme, and label. Spin direction is controlled globally by the shared asset layer.

Consumption patterns

Pages request instruments by contract.

Direct globe box Use inside a hero or panel.

Mount a globe where the page needs a specific object.

<div data-dgb-globe="earth" data-dgb-globe-size="hero" data-dgb-globe-theme="blue" data-dgb-globe-label="Earth standard" ></div>
Auto stage Let the runtime build the globe.

Use a stage when the page needs a full visual instrument section.

<div data-dgb-globe-stage data-dgb-globe-size="panel" data-dgb-globe-theme="gold" data-dgb-globe-label="Shared globe" ></div>
Diamond controls Use shared route classes.

Route cards and navigation bubbles inherit diamond shape and motion through the shared CSS.

<a class="route-card" href="/frontier/"> <small>Edge</small> <strong>Frontier</strong> </a>
