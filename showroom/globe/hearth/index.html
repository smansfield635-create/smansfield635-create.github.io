Cycle governance

        
The runtime moves in cycles.

        

          The route does not run as a single overloaded file. East starts, West admits, North governs,
          South completes, and the checkpoint decides whether the cycle passes, warns, retries, queues,
          degrades, or blocks.
        


        

          

            East
            Step 1 ignition, first paint, mount certainty, early ledger.
          

          

            West
            Handoff admissibility, receipt normalization, malformed packet rejection.
          

          

            North
            Route-local command runtime table, NEWS/Fibonacci checkpoint law.
          

          

            South
            Visible completion, canvas proof, inspect truth, final diagnostic receipt.
          

          

            Checkpoint
            Pass, warn, queue, retry, degrade, or hard-block only on kill conditions.
          

        

      


      

        

          
Survival standard

          
The world must remain inspectable.

          

            Hearth is not a blank planet and not a decorative globe. It is the survival proof lane:
            a visible Earth-condition field that must remain readable while the system tests coherence,
            load, motion, land, water, atmosphere, and inspection.
          

        


        

          
Diagnostic boundary

          
The cockpit is an instrument.

          

            The cockpit may report progress, copy diagnostics, and show receipts. It must also leave
            the planet frame when inspection is required. The user must be able to view Hearth without
            the diagnostic layer blocking the world.
