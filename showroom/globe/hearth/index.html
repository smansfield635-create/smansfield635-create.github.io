About Hearth
Hearth gives survival a visible condition.

Hearth is not a decorative planet page. It is the survival-path field where the visible planet, inspection controls, completion latch, and route diagnostics must survive together.

The page respects browser load supremacy: no in-page diagnostic bar is authoritative while the browser itself is still loading.

Runtime Cycle
The runtime entry attaches after page load.

HTML loads the runtime table, source stack, canvas authority, and East entry only. East then loads West, North, and South. This prevents duplicate downstream script loading.

East starts the visible process. West normalizes handoff admissibility. North governs command runtime-table checkpoints. South consumes the shell and proves visible completion.
