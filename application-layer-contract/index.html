Application Layer Rules

1. Definition

The Application Layer is a read-only projection surface. It does not define truth, admissibility, or authority.

2. Allowed Operations

  • Display coherence states
  • Visualize trajectories
  • Expose stability and fracture
  • Render time-based change

3. Prohibited Operations

  • Redefine laws or principles
  • Override admissibility
  • Create executive decisions
  • Introduce competing truths

4. Data Direction

UCAQ → M256 → EAI → Application Layer

5. Violation

Any component that violates this contract is non-canonical and may be removed without discussion.

Status
This contract is binding for all application-layer artifacts.
