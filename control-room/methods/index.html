Methods

This page defines the operational rules used to build and verify the system. Methods are constraints, not narratives.

Foundational Laws

  • Authority Law — Execution occurs only in the Control Room.
  • Observer Law — Dashboards read state; they never write.
  • Prefix Law — Engine discovery is defined by ENGINE::.
  • Filesystem Law — URL == folder == index.html.

Execution Geometry

  • Quad-Core Fusion — Selection, Resolution, Commitment, Continuation.
  • Namespace — Stable identifier per engine instance.
  • Registry — Device-local discovery via prefix scan.

Operational Procedure

  • Engines register by executing and writing ENGINE:: keys.
  • Matrix discovers engines via prefix scan.
  • Supervisor declares run context (labels/modes).
  • Observers summarize state without mutation.

Validation

Validation is structural. A method is valid if state is discoverable, isolated, and reproducible on the same device without external dependencies.
