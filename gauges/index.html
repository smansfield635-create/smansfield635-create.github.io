Products problem isolation

Find the visible stage owner.

This gauges pass is dedicated to one question only: which layer is actually producing the public-facing output on /products/, and where redundancy is creating the blank blue field and blob behavior. The goal is to isolate ownership, not decorate the surface.

Stage ownership map

Host layer · /products/index.html

Owns protected host copy, gateway framing, status container, and any primary static presentation shell. If this layer is also trying to present the actual product stage, it competes with downstream runtime ownership.

Candidate owner
Bootstrap layer · /products/index.js

Owns handoff, staging rules, mount targeting, and any injected presentation generated after boot. If this layer is also rendering a visible products experience while the runtime mounts underneath, redundancy emerges.

Primary suspect
Runtime layer · /products/products_runtime.js

Owns mounted runtime output and any visual field it paints into the stage. If it is still rendering visible proof or presentation while host/bootstrap render another surface, it creates stacked ownership.

Potential overlap
Visible field result

The live browser only gets one true public-facing stage owner. If more than one layer believes it owns visible stage output, the result collapses into blank backgrounds, residual blobs, or partially obscured surfaces.

One owner only
Current controlling read: the likely redundancy is not extra random files. It is competing stage ownership across index.html, index.js, and products_runtime.js.

Execution probes

Probe 1

Does index.html create a public-facing experience shell that already looks like the final product surface?

Probe 2

Does index.js inject another full products experience on top of that shell after successful boot?

Probe 3

Does products_runtime.js still paint its own visual stage underneath, even though bootchain and containment are already passed?

Probe 4

If the answer to more than one of the above is yes, the redundancy is proven and one visible-stage owner must be deleted or demoted.

Next lawful motion after this page: freeze products, identify the single visible stage owner, and remove redundant stage ownership cleanly instead of continuing to guess inside the products surface.

Direct audit lanes

Lane A Audit /products/index.html for host-owned visible stage markup that should remain shell-only.
Lane B Audit /products/index.js for secondary stage construction that duplicates what runtime is already mounting.
Lane C Audit /products/products_runtime.js for any retained visible proof surface that should become hidden, toggled, or non-primary.
Lane D Choose one public-facing owner only, then delete the competing presentation layer rather than layering more containment around it.
