Reference

Static documentation. Definitions, invariants, and terminology used across the system.

Core Terms

  • Engine — A stateful execution unit identified by a namespace.
  • Registry — Device-local discovery of engines via ENGINE:: keys.
  • Control Room — Authoritative surface for execution and supervision.
  • Engine Room — Workspace where engines execute and register.
  • Matrix — Read-only registry view of discovered engines.

Geometry

  • Quad-Core Fusion — Selection, Resolution, Commitment, Continuation.
  • Namespace — Stable identifier for an engine instance.
  • Prefix Law — Discovery is defined by string prefix (ENGINE::).
