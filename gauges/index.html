Definition layer

Runtime definitions are now explicit.

Physics model The runtime does not hand-design planetary order.

Physical constants define the relationship field: radius, orbital distance, orbital period, rotation period, and axial tilt. The website only translates those relationships into a readable inspection plane.

PHYSICS_MODEL=RELATIONSHIP_SOURCE BODY_FIELDS=RADIUS_KM,DISTANCE_AU,ORBITAL_DAYS,ROTATION_DAYS,AXIAL_TILT_DEG ORDER=PHYSICAL_ORDER_PRESERVED DESIGN=PROJECTION_NOT_PREMADE_UNIVERSE
Projection model The spread stays; the camera becomes cinematic.

The Sun is anchored on the left edge. Orbital lanes sweep diagonally across the scene. Distance is logarithmically compressed so the system remains visible without collapsing the planet spread.

PROJECTION_MODEL=CINEMATIC_SOLAR_SWEEP SUN_ANCHOR=LEFT_EDGE ORBIT_PLANE=DIAGONAL DISTANCE_MODEL=LOG_COMPRESSED SPREAD=KEEP COLLAPSE_PLANETS=FALSE
Celestial detail model Detail is coded into the bodies.

The runtime now adds surface detail directly to each body: solar turbulence, Mercury crater cues, Venus cloud bands, Earth land/cloud cues, Mars surface variation, Jupiter bands, Saturn rings, and outer-planet atmospheric shading.

DETAIL_MODEL=CODE_RENDERED SUN=CORONA,TURBULENCE,RADIAL_BANDING EARTH=LAND_CUES,CLOUD_CUES,RIM_SHADING JUPITER=BANDS,STORM_CUE SATURN=RING_DEPTH,BANDS MARS=SURFACE_VARIATION OUTER_PLANETS=ATMOSPHERIC_GRADIENTS
Protection rules No generated graphic path is allowed here.

The page remains a live coded runtime. It does not use GraphicBox, image generation, or static replacement art. The detail upgrade is handled through `/gauges/index.js`.

GRAPHIC_BOX=FORBIDDEN IMAGE_GENERATION=FORBIDDEN STATIC_IMAGE=FORBIDDEN RUNTIME=/gauges/index.js HOST=/gauges/index.html DETAIL_SOURCE=CSS_JS_ONLY
